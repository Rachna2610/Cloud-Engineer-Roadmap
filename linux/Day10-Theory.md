# Day 10 - Log Management and Monitoring

## Objective

Learn how Linux stores system logs, monitor system resources, and troubleshoot server issues using log files and monitoring commands.

---

# What are Logs?

Logs are records of events that occur in a Linux system.

Whenever a user logs in, a service starts or stops, software is installed, or an error occurs, Linux records the event in log files.

Logs are essential for troubleshooting and monitoring servers.

---

# Why are Logs Important?

Logs help administrators and Cloud Engineers to:

- Identify system errors
- Troubleshoot server issues
- Monitor applications
- Track user activities
- Detect security events
- Analyze system performance

Without logs, it is difficult to find the cause of a problem.

---

# Linux Log Directory

Most Linux log files are stored in:

```bash
/var/log
```

View the log directory:

```bash
ls /var/log
```

---

# Common Log Files

## 1. System Log

```bash
/var/log/syslog
```

Contains general system messages and events.

---

## 2. Authentication Log

```bash
/var/log/auth.log
```

Contains:

- User login attempts
- SSH logins
- Authentication events

---

## 3. Kernel Log

```bash
/var/log/kern.log
```

Contains messages generated by the Linux kernel.

---

## 4. Boot Log

```bash
/var/log/boot.log
```

Contains information generated during system startup.

---

# Viewing Log Files

## View Entire Log

```bash
cat /var/log/syslog
```

Displays the complete log file.

---

## View Log Page by Page

```bash
less /var/log/syslog
```

Useful for large log files.

Navigation:

- Space → Next page
- b → Previous page
- q → Quit

---

## View Last Lines

```bash
tail /var/log/syslog
```

Displays the last 10 lines.

---

## Monitor Logs in Real Time

```bash
tail -f /var/log/syslog
```

Displays new log entries as they are generated.

Stop using:

```text
CTRL + C
```

---

# journalctl Command

Modern Linux systems use **systemd**, which stores logs in a binary journal.

View all logs:

```bash
journalctl
```

---

## View Recent Logs

```bash
journalctl -n 20
```

Displays the last 20 log entries.

---

## View Logs for a Service

Example:

```bash
journalctl -u cron
```

Displays logs related to the cron service.

---

## View Today's Logs

```bash
journalctl --since today
```

Displays today's log entries.

---

# dmesg Command

Displays Linux kernel messages.

Command:

```bash
dmesg
```

Useful for:

- Hardware detection
- Boot messages
- Driver information

---

# Monitoring System Resources

Monitoring helps identify performance problems before they affect applications.

---

## Memory Usage

Command:

```bash
free -h
```

Displays:

- Total memory
- Used memory
- Free memory
- Swap memory

---

## Disk Usage

Command:

```bash
df -h
```

Displays:

- Disk size
- Used space
- Available space
- Mounted filesystems

---

## Directory Size

Command:

```bash
du -sh
```

Example:

```bash
du -sh Downloads
```

Displays the size of a directory.

---

## CPU and Process Monitoring

Command:

```bash
top
```

Displays:

- CPU usage
- Memory usage
- Running processes
- System load

Press:

```text
q
```

to quit.

---

# Real Cloud Engineer Example

Suppose an application is not working on an AWS EC2 instance.

A Cloud Engineer may follow these steps:

1. Check whether the service is running.

```bash
systemctl status nginx
```

2. View service logs.

```bash
journalctl -u nginx
```

3. Check available memory.

```bash
free -h
```

4. Check available disk space.

```bash
df -h
```

5. Monitor running processes.

```bash
top
```

These commands help identify and resolve server issues quickly.
